一、设计系统基础

当前CSS缺乏统一的设计Token体系，大量硬编码值散落在各处，导致维护成本高、一致性差。

P0 缺少统一的 Design Token 系统 ▼

当前 :root 中只定义了少量CSS变量（--bg-color, --primary 等），但大量颜色、间距、圆角值以硬编码方式散布在2800行CSS中。

问题示例：

/* 当前：大量硬编码的 rgba 值 */
.glass { background: rgba(17, 24, 39, 0.85); }
.btn-small { border: 1px solid rgba(255,255,255,0.06); }
.form-section { background: rgba(15, 23, 42, 0.22); }
/* 至少 30+ 处不同的 rgba 值 */

建议修复 建立完整的Token层级：

:root {
  /* 基础色板 */
  --slate-50: #f8fafc;
  --slate-900: #0f172a;
  
  /* 语义化Token */
  --surface-primary: rgba(17, 24, 39, 0.85);
  --surface-elevated: rgba(15, 23, 42, 0.4);
  --border-subtle: rgba(255, 255, 255, 0.06);
  --border-default: rgba(255, 255, 255, 0.1);
  --border-strong: rgba(255, 255, 255, 0.15);
  
  /* 间距 */
  --space-xs: 0.25rem;
  --space-sm: 0.5rem;
  --space-md: 1rem;
  --space-lg: 1.5rem;
  --space-xl: 2rem;
  
  /* 圆角 */
  --radius-sm: 6px;
  --radius-md: 10px;
  --radius-lg: 14px;
  --radius-xl: 16px;
}
P1 圆角值不统一 (4px / 5px / 6px / 8px / 10px / 12px / 14px / 16px) ▼

CSS中使用了至少 8种不同的圆角值，缺乏统一的圆角比例系统：

圆角值使用位置数量
4px.tag-badge, .status-badge~5处
5px.status-btn, .delete-btn~3处
6px.settings-clickable, .totp-group-tag~6处
8px.btn-small, .tag-list-item~8处
10px.btn, .sort-select, .stat-badge~10处
12px.history-item, .form-section, .details~12处
14px.totp-card, .totp-sidebar, .child-item~5处
16px.glass, .main-nav~6处

建议修复 统一为4级圆角系统：--radius-sm: 6px, --radius-md: 10px, --radius-lg: 14px, --radius-xl: 16px

P1 浅色主题覆盖过于碎片化 ▼

浅色主题通过 [data-theme="light"] 选择器逐个覆盖样式，当前已有 60+ 条 覆盖规则，维护成本极高且容易遗漏。

更好的做法是使用语义化Token，让主题切换只改变Token的值：

/* 当前：60+ 条逐个覆盖 */
[data-theme="light"] .history-name { color: #1e293b; }
[data-theme="light"] .history-meta { color: #64748b; }
[data-theme="light"] .totp-label { color: #1e293b; }
/* ... 还有 57 条 ... */

/* 建议：只切换Token值 */
:root { --text-primary: #f1f5f9; }
[data-theme="light"] { --text-primary: #1e293b; }
/* 所有使用 --text-primary 的地方自动适配 */

二、可访问性 (Accessibility)

作为涉及财务数据的应用，可访问性尤其重要。当前存在多个WCAG AA合规问题。

P0 多处颜色对比度不足 (WCAG AA 违规) ▼

WCAG AA要求正文对比度 ≥ 4.5:1，大文字 ≥ 3:1。当前多处不达标：

元素前景色背景色对比度状态
.text-secondary / .nav-btn #94a3b8 #0a0e1a 5.2:1 达标
.text-muted / .badge-label #64748b #0a0e1a 3.1:1 不达标
disabled button opacity: 0.5 varied ~2.5:1 不达标
.totp-remaining #64748b rgba(255,255,255,0.03) ~2.8:1 不达标

建议修复 将 --text-muted 从 #64748b 调整为 #8494a7（对比度提升至4.6:1），或在深色背景上使用更亮的灰色。

P0 缺少焦点指示器 (Focus Indicator) ▼

键盘用户无法通过Tab键看到当前焦点位置。CSS中 没有定义任何 :focus-visible 样式，浏览器默认的outline被 outline: none 移除后没有任何替代。

/* 当前：焦点样式被移除 */
input:focus, select:focus {
  outline: none; /* ← 键盘用户看不到焦点 */
  border-color: var(--primary);
  box-shadow: 0 0 0 3px var(--primary-glow);
}

/* 按钮完全没有焦点样式 */
.btn { /* 没有 :focus 规则 */ }

建议修复 为所有交互元素添加 :focus-visible 样式：

.btn:focus-visible {
  outline: 2px solid var(--primary);
  outline-offset: 2px;
}
P1 触摸目标尺寸不足 (Touch Target) ▼

WCAG 2.5.5 要求交互元素最小触摸目标为 44×44px。以下元素不达标：

元素实际尺寸问题
.delete-btn / .status-btn~32×32px仅 padding:8px + font-size:1.1rem
.btn-icon~24×28pxpadding: 2px 6px
.totp-delete-btn~28×28pxpadding: 2px 6px
.toggle-children-btn~60×22px高度不足
.modal-close~32×32px纯文字×，无padding

建议修复 将所有交互元素的最小点击区域设为 44×44px，可通过增加 min-height 和 min-width 实现。

P1 缺少ARIA标签和语义化标记 ▼

多个交互组件缺少必要的ARIA属性：

  • FAB按钮 + 没有 aria-label
  • Modal弹窗缺少 role="dialog" 和 aria-modal="true"
  • Tab切换组件缺少 role="tablist" / role="tab" / aria-selected
  • 统计badge缺少 aria-label 描述
  • 树形展开/收起按钮缺少 aria-expanded

建议修复 为所有交互元素添加语义化ARIA属性，确保屏幕阅读器能正确解读界面。

三、交互与用户体验

交互细节直接影响用户的操作效率和满意度。

P0 按钮层级混乱 — 所有按钮视觉权重相同 ▼

当前 .btn 类默认是全宽、渐变背景、大字号的"主按钮"样式。但"取消"、"返回"等次要操作也使用相同样式，导致视觉层级混乱。

当前状态

保存修改
取消
返回

所有按钮都是全宽渐变，用户无法区分主次

建议方案

保存修改
取消
返回

主要操作突出，次要操作弱化

建议修复 建立三级按钮系统：Primary（渐变填充）、Secondary（边框/透明）、Ghost（纯文字），并明确各自的使用场景。

P1 表单输入框缺少占位符样式对比 ▼

输入框的 placeholder 颜色与实际输入文字颜色对比度太低（都是白色系），用户难以区分"已填写"和"未填写"状态。

建议修复 为 placeholder 设置更明显的弱化色：

input::placeholder {
  color: rgba(148, 163, 184, 0.5);
  font-style: italic;
}
P1 Modal弹窗缺少入场/退场动画 ▼

Modal关闭时直接 display:none，没有退场动画，体验生硬。当前只有入场的 slideUp 动画。

建议修复 添加关闭动画，使用 opacity + transform 过渡：

.modal-overlay.closing {
  opacity: 0;
  pointer-events: none;
  transition: opacity 0.2s ease;
}
.modal-overlay.closing .modal-content {
  transform: translateY(20px) scale(0.95);
  transition: transform 0.2s ease;
}
P2 统计面板的图表标题位置不一致 ▼

成本排名图表的标题使用 writing-mode: vertical-rl 竖排显示在左侧，而趋势图标题在底部居中，标签分布图标题在顶部。三种图表三种标题位置，缺乏一致性。

建议修复 统一图表标题位置为顶部左对齐或底部居中，保持视觉节奏一致。

四、响应式设计

移动端适配整体不错，但存在一些边界情况。

P1 断点系统不统一 (768px vs 799px vs 600px vs 480px) ▼

CSS中使用了4个不同的断点值，缺乏统一的断点系统：

@media (max-width: 480px)   /* nav字体 */
@media (max-width: 600px)   /* form-row */
@media (max-width: 768px)   /* 大量移动端样式 */
@media (max-width: 799px)   /* mobile nav, body padding */
@media (min-width: 800px)   /* desktop layout */

建议修复 统一为3个标准断点：sm: 640px, md: 768px, lg: 1024px

P2 FAB按钮在桌面端可能遮挡内容 ▼

FAB按钮固定在右下角 position: fixed; bottom: 30px; right: 30px，在桌面端可能遮挡列表底部内容。

建议修复 在桌面端考虑将添加按钮集成到列表头部，或为列表底部增加 padding-bottom 避免遮挡。

五、视觉打磨

P2 毛玻璃效果过度使用 ▼

.glass 类被应用到几乎所有容器上（卡片、导航、Modal、侧边栏），backdrop-filter: blur(16px) 在低端设备上会造成性能问题，且过度使用会降低视觉层次感。

建议修复 只在顶层容器使用毛玻璃效果，子容器使用纯色背景。或者检测设备性能，低性能设备自动降级。

P2 滚动条样式仅支持WebKit ▼

自定义滚动条使用 ::-webkit-scrollbar 伪元素，Firefox用户会看到默认滚动条。

建议修复 添加标准属性 scrollbar-width: thin 和 scrollbar-color：

.history-list {
  scrollbar-width: thin;
  scrollbar-color: rgba(255,255,255,0.2) transparent;
}
P2 缺少空状态的视觉引导 ▼

空状态（无记录、无标签数据）只显示纯文字，缺少图标和行动引导。新用户首次进入时不知道如何开始。

建议修复 添加空状态插图和明确的CTA按钮：

整体评价：DayCost 的 UI 已经具备了不错的基础——Financial Noir 风格的暗色主题辨识度高，玻璃态卡片有质感，移动端适配做了不少工作。但设计系统层面的缺失（Token、一致性）和可访问性问题是阻碍其走向"商业级"的主要瓶颈。建议按优先级矩阵，先解决高影响低成本的问题，再逐步构建完整的设计系统。
